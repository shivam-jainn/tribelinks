# Implementation Plan Formatting Guidelines

## Target Path
- Save the plan to: `<appDataDir>/brain/<conversation-id>/implementation_plan.md`

## Structure
The document must follow the standard planning template:
1. **Goal Description**: Brief description of what the change accomplishes.
2. **User Review Required**: Highlight breaking changes, critical design decisions, and warnings.
3. **Open Questions**: Clarifying or design questions.
4. **Proposed Changes**: Grouped by component/package, specifying exact files to modify, create, or delete with links.
5. **Verification Plan**: Automated tests and manual steps.
